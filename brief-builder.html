Overview

What I built and why

As a recent marketing graduate, I noticed that writing campaign briefs takes marketers and business owners hours of work — even though the structure is largely the same every time. I wanted to build a tool that could generate a complete, professional campaign brief in seconds, based on a few guided inputs.

The twist: I had zero coding experience going in. This project was as much about learning how modern AI tools enable non-developers to build real software as it was about the marketing problem itself.

Problem & solution

The brief process is broken for small teams

The problem

Writing a campaign brief manually takes 2–4 hours. Most templates are either too generic to be useful or too complex for small business owners without marketing training. The result: campaigns start without alignment, goals are vague, and budgets get wasted.

The solution

A guided 4-step form that takes 3 minutes to fill in. An AI then generates a complete, structured brief — objective, key message, channel strategy, KPIs, budget split, and timeline — instantly. No marketing degree required.

The real build process

Every mistake, and what I learned from it

This is the part most portfolios leave out. Here's exactly what went wrong during the build — and how I worked through each problem.

✕
Mistake 1 Uploaded zip files to GitHub instead of extracted files

GitHub needs actual files — not a compressed archive. I uploaded the zip without extracting it first, so the repo contained .zip files instead of index.html and the function folder.

❌ campaign-brief-builder.zip uploaded directly to repo
✓
Fix 1 Extracted the zip and uploaded the folder structure correctly

Deleted the zip files, extracted locally, then dragged the folder into GitHub's upload interface. Learned: GitHub reads files, not archives.

✓ index.html / netlify.toml / netlify/functions/generate-brief.js
✕
Mistake 2 Pasted the API key as the variable name, not the value

In Netlify's environment variables panel, I put the actual API key string (AIzaSy...) in the Key name field instead of the Value field. The variable had no actual value set.

Key: AIzaSyB_iJuKK... | Value: (empty)
✕
Mistake 3 Typo in the environment variable name

After fixing the value, the variable name was still wrong — I typed GEMENI_API instead of GEMINI_API_KEY. The code was looking for an exact name that didn't exist, so the key was invisible to the function.

❌ GEMENI_API → ✓ GEMINI_API_KEY
✕
Mistake 4 The AI model had been deprecated by Google

The original code used gemini-1.5-flash, but Google's API had changed its model names. Then I switched to gemini-2.0-flash — which turned out to have been officially retired in March 2026 with a free tier limit of zero requests.

Error 404: models/gemini-1.5-flash is not found for API version v1beta
✕
Mistake 5 JSON parsing failed on the AI's response

Once the correct model (gemini-2.5-flash) was working, the response came back but the code couldn't parse it. The newer model wraps its output differently, causing an "Unterminated string" error on line 65.

SyntaxError: Unterminated string in JSON at position 601
★
Final fix Robust JSON extraction + correct model = working app

Added responseMimeType: "application/json" to force clean output, and rewrote the parser to find the JSON boundaries rather than assuming perfect formatting. First successful brief generated.

✓ gemini-2.5-flash / v1beta / responseMimeType: application/json
